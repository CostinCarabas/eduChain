3

Emitere certificat NFT (SBT) pentru student

Emite un certificat Soulbound Token cu hash SHA-256 al metadatelor stocat imutabil on-chain

▾

Flux de emitere certificat

1
Citire fixture JSON (cert1.json) — date student, curs, instituție
2
Citire metadata JSON-LD off-chain și canonicalizare (chei sortate)
3
Calcul SHA-256 al JSON-LD canonic → content_hash unic
4
Construire CertificateAttributes cu content_hash + date student
5
Trimitere tranzacție issueCertificate pe Devnet → confirmare blockchain
6
NFT (SBT) transferat automat în wallet-ul studentului
# Inițializare colecție NFT (prima dată): rust-interact nft issue-token --name EduCert --ticker EDUCERT rust-interact nft set-local-roles rust-interact nft add-issuer --addr <operator> # Emitere certificat din fixture: rust-interact nft issue --cert-file fixtures/cert1.json # Verificare atribute on-chain: rust-interact nft verify --nonce 1
4

Distribuire recompense $ECT către student

Adaugă recompense ECT pentru student (absolvire) și permite claim-ul fondurilor

▾

Flux recompense

1
Înregistrare operator ca distribuitor autorizat de recompense
2
Adaugă 500 ECT în pending pentru adresa studentului (motiv: "absolvire")
3
Verifică soldul pending al studentului on-chain
4
Studentul execută claim → ECT transferați în wallet
rust-interact token add-distributor --addr <operator> rust-interact token add-reward --to <student> --amount 500 --reason "absolvire" rust-interact token pending-balance --address <student> rust-interact token claim
5

Sesiune de mentoring cu escrow $ECT

Student plătește 200 ECT blocați în escrow — eliberați mentorului după confirmare

▾

Flux escrow mentoring

1
Student creează sesiunea → 200 ECT blocați în smart contract escrow
2
Sesiunea rămâne în stare "Open" pe durata mentoring-ului (deadline: 72h)
3
Student confirmă finalizarea sesiunii on-chain
4
200 ECT eliberați automat în wallet-ul mentorului → stare "Completed"
rust-interact escrow set-token-id rust-interact escrow create --mentor <mentor_addr> --amount 200 --deadline-hours 72 rust-interact escrow get-session --id 0 rust-interact escrow confirm --id 0 rust-interact escrow get-session --id 0
6

Ancorare hash document pe blockchain

Calculează SHA-256 al raportului de validare TRL-4 și îl stochează imutabil on-chain

▾

Flux ancorare

1
Calcul SHA-256 al fișierului local (EduChain_TRL4_E2_Validation_Report.md)
2
Trimitere hash (32 bytes) + URI pe blockchain — stocat cu autor, timestamp
3
Verificare ancorei pe blockchain (hash → metadata)
4
Test duplicat → contractul respinge anchore duplicate (imutabilitate)
rust-interact anchor put --file ../docs/EduChain_TRL4_E2_Validation_Report.md # → Hash SHA-256: a3f8c2... stocat on-chain cu timestamp rust-interact anchor verify --hex <sha256_hash> # → anchor_contract returnează: autor, timestamp, URI, revoked=false # Test de idempotență (trebuie să eșueze): rust-interact anchor put --file ../docs/EduChain_TRL4_E2_Validation_Report.md # ✔ Contractul a respins corect hash-ul duplicat!
7

Revocare certificat (soft revoke)

Marchează certificatul ca revocat on-chain — NFT rămâne în wallet, statusul devine "Revoked"

▾

Flux revocare

1
Emitent apelează revokeCertificate cu nonce + motiv
2
Smart contract actualizează statusul NFT-ului → "Revoked"
3
NFT rămâne în wallet-ul studentului (Soulbound — non-transferabil)
4
Orice verificare ulterioară returnează status=Revoked + motiv + timestamp
rust-interact nft revoke --nonce 1 --reason "date eronate" # ✔ Tranzacție confirmată pe Devnet rust-interact nft verify --nonce 1 # → status: Revoked | motiv: date eronate | timestamp: ...

⚙ Setup pentru Demo

⚠ Pași obligatorii înainte de demo

1
Configurează variabila PEM:
export EDUCHAIN_PEM="$(cat ~/wallets/devnet_operator.pem)"
Cheia privată a operatorului pe Devnet MultiversX (NU o commite în git!)
2
Compilează binarul interactor:
cd eduChain-PoC && cargo build --release -p rust-interact
Binarul va fi la: target/release/rust-interact
3
Verifică conexiunea Devnet:
curl https://devnet-gateway.multiversx.com/about
Trebuie să returneze info despre rețea (acces internet necesar)
4
Rulează demo-ul:
chmod +x demo/demo.sh
./demo/demo.sh — mod interactiv (cu pauze)
AUTO=1 ./demo/demo.sh a — mod automat, toți pașii
SKIP_DEPLOY=1 ./demo/demo.sh a — sare deploy (contracte deja existente)
5
Demo pas individual:
./demo/demo.sh 3 — rulează doar Pasul 3 (emitere certificat)
./demo/demo.sh 6 — rulează doar Pasul 6 (ancorare hash)
📋 Configurație curentă (state.toml)
# interactor/state.toml — adrese deployate pe Devnet: nft_contract = "erd1qqqqqqqqqqqqqpgqtgc2tn999va6cpmgujcv3jk4dt7wgam3lmpq6qakvu" token_contract = "erd1qqqqqqqqqqqqqpgqvyz832uc7pfan2usyqq8ek9k8vf7xhtvlmpqsf73v7" escrow_contract = "erd1qqqqqqqqqqqqqpgq08hh9e5t57llh8p300hfdneqlxj80shzlmpq4f6qd6" anchor_contract = "erd1qqqqqqqqqqqqqpgqcy8ws6k9fthwfa9g67ksynvcfen7rdmglmpqpc7cl7" ect_token_id = "ECT-f90acc" # interactor/config.toml: chain_type = "real" gateway_uri = "https://devnet-gateway.multiversx.com"
